# P10 — Employer PAYE Return (Section B)

Generates the KRA P10 **Section B** ("Details of Salary Paid and PAYE deducted
from Employees") import CSV for a payroll period, ready to load into the
official `P10_Return.xlsm` via its **IMPORT CSV** button on the
`B_Employees_Dtls` sheet.

## Endpoint

```
GET /payroll/p10?year=<YYYY>&month=<1-12>
```

- HR-management-role gated (`Admin`, `HR Manager`, `HR Officer`).
- Returns `text/csv` as an attachment (`P10-Section-B-<year>-<month>.csv`).
- Data rows only, **no header row** — iTax maps positionally and would treat a
  header as a data row. CRLF line endings.
- Includes every employee with a **finalized** payslip in the period. A period
  with no finalized run returns an empty body.

## Column format (24 fields, exact iTax order)

The order below is the canonical **import** order taken from the real template's
`ValidationList` — which is **not** the sheet's visual column order (e.g. Value
of Car Benefit is last in the CSV but appears mid-sheet). Verified against
`P10_Return-Version-22.0.0.xlsm`.

| # | Column | Source in our data |
|---|--------|--------------------|
| 1 | PIN of Employee | employee `kraPin` (decrypted) |
| 2 | Name of the Employee | first + last name |
| 3 | Residential Status | `Resident` (default) |
| 4 | Type of Employee | `Primary Employee` (default) |
| 5 | Basic Salary | structure basic |
| 6 | Housing Allowance | taxable allowance, name ~ house/rent/accommodation |
| 7 | Transport Allowance | taxable allowance, name ~ transport/travel/commute/fuel/mileage |
| 8–11 | Leave / Overtime / Director's Fee / Lump Sum | 0 (not modelled) |
| 12 | Other Allowance | remaining **taxable** allowances |
| 13 | Other Non cash Benefits | 0 |
| 14 | Global Income | 0 |
| 15 | Type of Housing | `Benefit not given` (default) |
| 16–17 | Rent of House / Rent Recovered | 0 |
| 18 | Actual Contribution | NSSF employee (the pension contribution) |
| 19–20 | Mortgage Interest / HOSP Deposit | 0 |
| 21 | Monthly Personal Relief | from the engine (2,400) |
| 22 | Self Assessed PAYE Tax | **our computed PAYE** (authoritative) |
| 23 | Amount of Insurance Relief | 0 (optional column) |
| 24 | Value of Car Benefit | 0 |

The spreadsheet **computes** the other 11 columns (Total Cash Pay, Taxable Pay,
Tax Payable, its own PAYE, etc.) from these inputs — the CSV supplies only the
inputs above.

### Only taxable cash pay is exported

Non-taxable allowances (e.g. per-diem) are **excluded** from the Section B cash
columns, so cash pay equals our taxable basis rather than gross. This keeps the
spreadsheet's taxable-pay computation aligned with ours on the cash side.

## PAYE reconciliation — Self-Assessed (AI) vs the template's computed PAYE (AH)

**Column AI (Self-Assessed PAYE) is the authoritative figure and is what we
populate.** The template's own computed PAYE (column AH) will typically read a
little **higher** than ours. This is expected and correct — do not "fix" it.

Why the gap exists: since **27 December 2024** (Tax Laws (Amendment) Act 2024),
SHIF and AHL are **allowable deductions before PAYE**. Our engine deducts
NSSF + SHIF + AHL from taxable pay. The v22.0.0 template's Section B, however:

1. has **no SHIF/AHL columns** and does not subtract them (its taxable pay =
   gross − pension benefit only); and
2. hardcodes the pension **permissible limit at 20,000/month**, whereas current
   law allows **30,000/month**.

So the template's built-in AH is a simplified / partly-outdated cross-check, not
the declared figure. KRA provides the separate **Self-Assessed PAYE** column
precisely so the employer can report their correctly-computed PAYE (AI); AH is
only a reference. The AH/AI difference affects every compliant Kenyan payroll
system today because the template lags the law.

### Worked example (Jane, gross 105,000; per-diem 5,000 non-taxable)

- Cash pay exported = 100,000 (basic 80,000 + housing 20,000; per-diem excluded)
- Actual Contribution (NSSF) = 6,300
- **Our taxable** = 100,000 − 6,300 − 2,887.50 (SHIF) − 1,575 (AHL) = 89,237.50
  → **PAYE (AI) = 19,154.60**
- **Template taxable (AD)** = 100,000 − 6,300 = 93,700
  → **template PAYE (AH) ≈ 20,493** (≈ 1,339 higher = 30% marginal × the
  SHIF+AHL that is legally deductible)

**Do not** map SHIF/AHL into "Actual Contribution" to force AH = AI: those are
not pension contributions, it would misreport the pension figure to KRA, and it
would break against the 20,000 cap. The self-assessed column is the correct
mechanism.
